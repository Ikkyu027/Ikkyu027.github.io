Hertz Rental System

A comprehensive web application designed for daily car rentals, offering a seamless experience for both customers and administrators.

Project Overview

The Hertz Rental system is a full-stack application aimed at streamlining the car rental process. It handles everything from vehicle listings and availability to user authentication and booking management. This project was developed to demonstrate skills in database management, backend logic, and responsive front-end design.

Key Features

  • User Authentication: User verification: Secure login and registration for borrowers, staff, and admin.
  • Vehicle Management: Admin can add, edit, and remove vehicle listings, including details like model, year, price, and availability.
  • Booking System: Users can browse available cars, select rental dates, and submit booking requests.
  • Booking Tracking: Users can view their past and upcoming rentals. Admins can manage all bookings.
  • Search and Filter: Users can easily find cars based on various criteria (e.g., car type, price range).
  • Responsive Design: Optimized for use on desktops, tablets, and mobile devices.

Technologies Used

  • Frontend: HTML, CSS, JavaScript
  • Backend: Node.js (with Express.js for API development)
  • Database: MySQL
  • Version Control: Git/GitHub

Project Challenges & Solutions

One significant challenge was ensuring accurate booking data while preventing double bookings. This was resolved by using proper SQL transaction handling and real-time date availability validation on both the client and server sides. Additionally, designing a clear and responsive user interface across various devices required attention to layout and media queries, which was accomplished using custom CSS and thorough user testing to improve usability.
